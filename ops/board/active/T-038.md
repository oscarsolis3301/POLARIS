## Why
Five pre-release QA findings on the 5.15 kit, verified in a scratch install. One builder, two files.

1. commands.md truncation ‚Äî `brain_commands | head -n 80` (kit/ops/polaris:1741) caps the file, but
the `usage()` dump alone is ~75 lines, so the 8 effective-CONVENTIONS values printed after it are cut
(publish, express, stale_hours, test, build were missing). Per brain v1.1: print the values BEFORE
the help dump ‚Äî the cap may cut the help tail, never the values.

2. Post-done staleness ‚Äî `cmd_done` touches `.polaris/board-changed` (kit/ops/polaris:862) AFTER
seal's `brain_seal_refresh` (:1214, :1281) wrote the stamp, so the documented wave close
(seal ‚Üí run-verify ‚Üí done) always ends with doctor warning "brain is stale". Per brain v1.1: `done`
mirrors seal ‚Äî brain exists ‚Üí run the cheap incremental refresh after the touch; failure = one ‚öÝ
note, never a red done. No brain dir ‚Üí do nothing.

3. Land noise ‚Äî `git merge --squash -q feat/<ID>` (kit/ops/polaris:996) still prints "Squash commit
-- not updating HEAD" and "Automatic merge went well; stopped before committing as requested" on
git 2.53. Redirect that command's stdout only; stderr stays live so real failures surface. Express
shares the code path ‚Äî one fix covers both (clean-history v2.2).

4. Commit type map ‚Äî `cmd_task_commit_msg` maps feature‚Üífeat, bug‚Üífix, else chore
(kit/ops/polaris:905-909), so the advertised commit types `test` and `docs` are unreachable ‚Äî a
`type: test` task lands as chore. Add test‚Üítest and docs‚Üídocs; chore stays the fallback for
spike/missing/unknown (clean-history v2.2). Update the map line at kit/ops/MANUAL.md:129 to match.

5. MANUAL count ‚Äî "Build these 7 files" (kit/ops/MANUAL.md:227) contradicts "the same 6 files"
(:235). Pinned phrasing (brain v1.1): "7 entries ‚Äî 6 `.md` files + `.stamp`"; greenfield keeps
"the same 6 `.md` files, near-empty". Also update the freshness bullet (:239) to say done AND seal
auto-refresh, per finding 2.

Each fix carries the selftest drill that would have caught it; drill fail tokens are pinned in the
contracts (brain v1.1 drills 6-7 ¬∑ clean-history v2.2).
