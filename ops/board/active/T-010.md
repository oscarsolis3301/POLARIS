## Why
A model that only exists in the CLI is invisible: kit/CLAUDE.md's THE TOOL table is the first (and
for token discipline, only) thing most sessions read, MANUAL.md is the contract-mandated fallback
for harnesses that cannot execute commands, and INIT.md is where a fresh repo learns its
conventions. Without these, agents keep following the old --no-ff recipe and humans never learn
`history` exists. This documents the clean-history surface everywhere the contract names — command
table rows for land/seal/history/rollback, hand-runnable MANUAL recipes for land and seal, an INIT
history-model note with the clean-log alias, and the 5.12.0 CHANGELOG entry.

## Acceptance criteria (binary — each becomes a test; put the runnable ones in verify:)
- [ ] kit/CLAUDE.md: THE TOOL table gains rows for `land` / `seal` / `history` / `rollback` (one line each, matching the table's existing terseness) plus a one-line statement of the history model (task = one squash commit, sprint = one tagged merge, board chores filtered by `history`); file stays within its current size discipline.
