## Why this exists
Field report, 5.6.0, reporter's #1: `fm_list` (kit/ops/polaris:79) special-cases only the empty `[]`;
a populated inline list `depends_on: [T-001, T-002]` comes back as ONE literal bracketed item. So the
ready gate false-positives forever, `drift --strict` can never pass on such a board, and `done` would
splice `[...]` into MAP.md via map_delta (kit/ops/polaris:443). `templates/TASK.md` itself shows
`depends_on: []` — one keystroke from the shape that breaks.

## Acceptance criteria (binary — each becomes a test; put the runnable ones in verify:)
- [x] All four shapes parse exactly per ops/contracts/frontmatter-lists.md (inline scalar · `[]` · populated `[a, b]` · block list); comments/\r/whitespace stripped, empty items dropped.
- [x] `doctor --selftest` gains parse drills covering the four shapes, including that `depends_on: [A, B]` yields two items.
- [x] Existing selftest stays green — block lists and scalars behave byte-identically to today.
- [x] Bash 3.2 compatible, awk stays POSIX, no new dependencies.
