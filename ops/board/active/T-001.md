## Why this exists
`install.sh` is the highest-consequence file in the kit — it is the only code that touches a
stranger's repo — and today it is the ONLY code with no test a human or Builder can run before
pushing. `test:` (`doctor --selftest`) drills the CLI and never invokes the installer at all. The
drills exist, but only inside `.github/workflows/ci.yml`, which agents may not edit (RULES) and
nobody can run locally.

That gap is not theoretical. The `kit/` split shipped with two real installer bugs that CI caught and
a local run would have caught sooner: an `ops/*.md` glob that raked this repo's `CONVENTIONS.md` into
any target (which then looked like a live board, locking INIT out of it), and a CLAUDE.md block-writer
that nested a new marker pair on every update until `uninstall` could no longer delimit the block.

Extract those drills into a script that both CI and a Builder run. The script is the deliverable; CI
is rewired to call it by a human afterwards (`.github/` is a danger zone — see Notes).

## Acceptance criteria (binary — each becomes a test; put the runnable ones in verify:)
- [ ] `bash kit/ops/selftest-install.sh` exits 0 on a healthy tree, non-zero on any failed drill, and prints one clear line per drill.
- [ ] Runs entirely in throwaway dirs (`mktemp -d`), leaves the repo byte-identical, and needs no network.
- [ ] Drill: **fresh install** from a freshly packed zip into a temp repo that already has its own `CLAUDE.md` and its own `PreToolUse` hook → both survive, guard is merged, `CLAUDE.md` has exactly one BEGIN/END pair.
- [ ] Drill: **no board leaks** — the packed zip contains no `CONVENTIONS.md` / `MAP.md` / `SPRINT.md` / `RULES.tsv` / `board/`, and neither does the installed target. Target must still report "INIT has not run".
- [ ] Drill: **old-client path** — install by running `ops/install.sh` from the REPO ROOT (this is what `polaris update` does: it installs from the branch tarball's root, which is our *installation*). Same no-leak and one-marker-pair assertions.
- [ ] Drill: **live-board refresh** — install twice over the same target; the second run reports `live-board`, refreshes kit code, and leaves board/CONVENTIONS/MAP/SPRINT/RULES byte-identical.
- [ ] Drill: **uninstall** — `polaris uninstall --yes` removes `ops/`, the managed block and the guard hook, keeps the user's own `CLAUDE.md` content and hooks.
- [ ] Bash 3.2 compatible (no `mapfile`, no associative arrays) — macOS ships 3.2 and CI pins `/bin/bash` there.
- [ ] Windows/Git Bash safe: no `zip`/`unzip` dependency (use `python kit/ops/pack.py`), no GNU-only flags.
