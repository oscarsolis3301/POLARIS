## Why
Under the new model the commit message is GENERATED from the task file — so message quality is
decided at authoring time, not at merge time. Today's template has nowhere to put the rationale
(boards improvised `## Why this exists`) and no scope field, so a generated message would open
with a derived guess and an empty body. This adds the two capture points the contract pins —
optional `scope:` frontmatter and a `## Why` section — and makes the Planner author both per task
and the Builder keep Notes as the one-line "how" discoveries that land in the commit body.

## Acceptance criteria (binary — each becomes a test; put the runnable ones in verify:)
- [ ] kit/ops/templates/TASK.md gains optional `scope:` frontmatter (with a one-line comment: conventional-commit scope, Planner sets it) and a `## Why` section between the frontmatter and Acceptance criteria, with a one-line prompt of what junior-dev-grade means; existing fields byte-identical.
- [ ] kit/ops/roles/PLANNER.md: task-creation steps require authoring `## Why` + `scope:` per task (commit quality is planned in, not improvised at merge) — a short addition to the existing protocol list, no renumbering churn beyond the touched steps.
