## Why
Today one landed task costs ~3 board commits + N builder WIP commits + a merge, none of the
messages saying what was built or why — the graph is unusable and "roll back to a known-good
state" is effectively impossible. This task gives the CLI the whole model: `land` squashes a feat
branch into ONE commit whose message is generated from the task file, `seal` closes a sprint as
ONE tagged `--no-ff` merge, `history` shows the changelog view, `rollback` reverts a task or a
sprint safely. Squashing breaks every "is it merged?" ancestry gate (done, remote cleanup, sweep),
so those gates switch to the landed-commit record — the contract pins the exact rules.

## Acceptance criteria (binary — each becomes a test; put the runnable ones in verify:)
- [ ] `task-commit-msg` prints exactly the contract's format (type map · scope fallback · Why body incl. legacy `## Why this exists` · What changed from checkboxes · Notes filtered of ⛔/comments · Files line); pure, mutates nothing.
- [ ] `land <ID>` implements the contract: audit → squash → ONE commit with `[<ID>]` subject + `Landed-from:` trailer; refuses on $BASE; conflict → reset --hard + kickback; empty diff → reset --hard + die; leaves zero uncommitted state on every path.
- [ ] `seal [<date>]` implements the contract: preconditions (clean tree · branch exists · ≥1 non-chore(board) commit · tag free) · `Sprint <n> — <goal>` merge message with per-task bullets · lightweight `sprint/<n>` tag · push when a remote exists · conflict aborts.
- [ ] `history [--tasks <n>]` shows first-parent $BASE minus `chore(board):` subjects; --tasks drills into one sprint per the contract ranges.
- [ ] `rollback <ID|sprint/<n>>` reverts per the contract (landed: fast path, `[<ID>]` subject search, `-m 1` for a sprint); refuses off-$BASE or dirty; conflicted revert aborts.
- [ ] `done` accepts a squash landing via the contract's gate rules (subject-in-base, legacy feat-ancestry fallback), stamps `landed:` frontmatter, and its remote cleanup + `sweep`'s stray detection follow the contract's tip-equality rules.
- [ ] `doctor --selftest` drills the new model (land format · seal merge+tag · history filter · done-on-squash · legacy --no-ff path still green · both reverts apply) and stays green; dispatch + usage list all five commands.
- [ ] Bash 3.2 compatible, POSIX awk, no new dependencies, Windows/Git Bash safe.
