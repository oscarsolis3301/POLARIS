## Why this exists
`kit/ops/dashboard.py` is the one entry point with zero automated coverage — CI never launches it
(MAP § Unverified). A silent break ships to every user who runs `polaris dash`.

## Acceptance criteria (binary — each becomes a test; put the runnable ones in verify:)
- [ ] `bash kit/ops/selftest-dashboard.sh` exits 0 on a healthy tree, non-zero on any failed assertion, one clear line per drill, total runtime under ~10s.
- [ ] Starts `kit/ops/dashboard.py` on a NON-default port (never 7373 — a live `polaris dash` must be untouched), waits for readiness, and kills the server on EVERY exit path (trap) — no orphan process even when an assertion fails.
- [ ] Asserts `GET /` → 200 and `GET /state` → 200 with valid JSON whose `columns` has exactly the six keys, per ops/contracts/dashboard-http.md.
- [ ] HTTP client is python3 stdlib (no curl/wget dependency); Bash 3.2 compatible (no mapfile, no assoc arrays); Windows/Git Bash safe.
- [ ] Read-only: leaves the repo byte-identical; dashboard.py is context, NEVER edited.
